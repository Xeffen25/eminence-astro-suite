This page documents file-based outputs that Eminence Astro Suite does not support by design. The project prioritizes modern, standards-oriented metadata and does not plan to support legacy or insecure platform behaviors. This keeps the integration smaller, easier to maintain, and aligned with current browser capabilities.

Modern alternatives to these files are already implemented in the integration and should be used instead. You can find the recommended alternative of each unsupported file.

The files listed here are considered outdated or legacy conventions that are no longer widely used or have been superseded by modern standards.

## browserconfig.xml

`browserconfig.xml` is a configuration file used by Internet Explorer 11 and Windows 8/10 to define how a website's tile should appear when pinned to the Start screen.

### Reason

It is tied to legacy Microsoft tile behavior and obsolete browser conventions. Modern browsers and platforms have moved towards standards-based metadata.

### Recommended Alternative

Use standards-based metadata through:

- [Web app manifest](/integration/manifest)
- [Modern icon metadata](/integration/icons) used by current browsers and platforms

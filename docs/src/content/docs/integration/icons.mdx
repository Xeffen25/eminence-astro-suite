The icons integration resolves icon metadata at build time from a file-keyed configuration. Each key is an output file name, and each value defines generation behavior, head-tag emission, and optional manifest registration.

When `source` is provided, the integration merges your entries over a built-in default icon set. You can replace any default by reusing its key, or disable it by setting that key to `false`.

## Options

### `IconsOptions`

| option                                         | type                                                                                                                                                                            | required | description                                                                                                                                                                                                                                                 |
| ---------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `source`                                       | `string`                                                                                                                                                                        | Yes      | Global source image used for generated raster and `.ico` outputs unless a keyed entry sets its own `source`.                                                                                                                                                |
| `[filename.ico]`                               | `{ sizes: number[] \| false; tag?: Partial<IconTag> & Pick<IconTag, "rel">; manifest?: boolean \| ManifestIconOptions; source?: string; sharpOptions?: SharpOptions } \| false` | No       | Generates a multi-size `.ico` file. `sizes: false` skips file generation only. Setting the key to `false` disables the full entry (generation, head tag, and manifest). Link-tag sizes are never inferred for `.ico`; set `tag.sizes` explicitly if needed. |
| `[filename.svg]`                               | `{ tag?: Partial<IconTag> & Pick<IconTag, "rel">; manifest?: boolean \| ManifestIconOptions } \| false`                                                                         | No       | Metadata-only SVG entry. Explicit SVG keys do not generate files. If global `source` is SVG and `favicon.svg` is not defined, the integration auto-copies the source and prepends a favicon tag. Set `"favicon.svg": false` to disable that auto-copy.      |
| `[filename.<png\|jpg\|jpeg\|gif\|webp\|avif>]` | `{ size: number \| false; tag?: Partial<IconTag> & Pick<IconTag, "rel">; manifest?: boolean \| ManifestIconOptions; source?: string; sharpOptions?: SharpOptions } \| false`    | No       | Generates a raster file at the configured square size. `size: false` skips file generation only. Setting the key to `false` disables the full entry. Raster link-tag sizes are inferred as `NxN`.                                                           |

### `IconTag`

| field   | type                          | required | description                                                                                                                                       |
| ------- | ----------------------------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| `rel`   | `string`                      | Yes      | Required to emit a `<link>` tag. If `tag.rel` is omitted, the file can still generate and contribute to the manifest, but no head tag is emitted. |
| `href`  | `string`                      | No       | Ignored in build config. The integration always derives `href` from the file key and normalizes it with a leading `/`.                            |
| `sizes` | `string`                      | No       | Optional explicit `sizes`. Used as-is for `.ico` or SVG metadata. Raster files infer this automatically from `size`.                              |
| `type`  | `string`                      | No       | Optional MIME override. Defaults to `inferImageMimeType(fileName)`.                                                                               |
| `media` | `"light" \| "dark" \| string` | No       | Optional media hint consumed by the runtime `Icons` component.                                                                                    |

### `ManifestIconOptions`

| field     | type     | required | description                                                                                                                 |
| --------- | -------- | -------- | --------------------------------------------------------------------------------------------------------------------------- |
| `src`     | `string` | No       | Override the manifest icon path. Defaults to the normalized file key.                                                       |
| `sizes`   | `string` | No       | Override inferred sizes. Raster icons infer `NxN`; `.ico` icons infer a space-separated list such as `"16x16 32x32 48x48"`. |
| `type`    | `string` | No       | Override inferred MIME type.                                                                                                |
| `purpose` | `string` | No       | Optional manifest purpose such as `"maskable"` or `"monochrome"`.                                                           |

## Examples

### Basic

Input:

```ts
eminence({
  icons: {
    source: "src/assets/logo.png",
  },
});
```

Output:

```html
<link
  rel="icon"
  href="/favicon.ico"
  sizes="16x16 32x32 48x48"
  type="image/x-icon"
/>
<link rel="icon" href="/favicon.png" sizes="32x32" type="image/png" />
<link
  rel="apple-touch-icon"
  href="/apple-touch-icon.png"
  sizes="180x180"
  type="image/png"
/>
<link rel="icon" href="/icon-192.png" sizes="192x192" type="image/png" />
<link rel="icon" href="/icon.png" sizes="512x512" type="image/png" />
```

### Automatic

When `source` is SVG and `favicon.svg` is not explicitly configured, the source is auto-copied to `/favicon.svg` and its tag is prepended.

Input:

```ts
eminence({
  icons: {
    source: "src/assets/logo.svg",
  },
});
```

Output:

```html
<link rel="icon" href="/favicon.svg" sizes="any" type="image/svg+xml" />
<link
  rel="icon"
  href="/favicon.ico"
  sizes="16x16 32x32 48x48"
  type="image/x-icon"
/>
<link rel="icon" href="/favicon.png" sizes="32x32" type="image/png" />
<link
  rel="apple-touch-icon"
  href="/apple-touch-icon.png"
  sizes="180x180"
  type="image/png"
/>
<link rel="icon" href="/icon-192.png" sizes="192x192" type="image/png" />
<link rel="icon" href="/icon.png" sizes="512x512" type="image/png" />
```

### Complete

Input:

```ts
eminence({
  icons: {
    source: "src/assets/logo.svg",
    "favicon.ico": false,
    "apple-touch-icon.png": {
      size: 180,
      tag: { rel: "apple-touch-icon" },
      manifest: true,
    },
    "badge.png": {
      size: 96,
      tag: { rel: "icon" },
      manifest: { src: "/brand/badge.png", purpose: "monochrome" },
    },
    "safari-pinned-tab.svg": {
      tag: { rel: "mask-icon", color: "#111111" },
    },
  },
});
```

Output:

```html
<link rel="icon" href="/favicon.svg" sizes="any" type="image/svg+xml" />
<link rel="icon" href="/favicon.png" sizes="32x32" type="image/png" />
<link
  rel="apple-touch-icon"
  href="/apple-touch-icon.png"
  sizes="180x180"
  type="image/png"
/>
<link rel="icon" href="/icon-192.png" sizes="192x192" type="image/png" />
<link rel="icon" href="/icon.png" sizes="512x512" type="image/png" />
<link rel="icon" href="/badge.png" sizes="96x96" type="image/png" />
<link
  rel="mask-icon"
  color="#111111"
  href="/safari-pinned-tab.svg"
  type="image/svg+xml"
/>
```

## Decisions Made

### File keys define public URLs

Config keys are output file names, so generated URLs, head tags, and manifest icons come from one source of truth.

### `false` disables the full entry

Setting a file key to `false` disables generation and metadata output for that key. This differs from `size: false` or `sizes: false`, which skip generation only.

### Merge precedence is explicit

Resolution layers in this order: generated entries from `icons`, then `headTags.icons` by `href`, then `manifest.icons` by `src`. Later layers override earlier collisions.

### SVG auto-favicon is conditional

`favicon.svg` is auto-added only when `source` is SVG and the user did not define or explicitly disable `favicon.svg`.

---
title: Icons - Component
description: Documentation for Icons, focused on rendering resolved icon link tags. It accepts late-bound overrides that merge over the build-time icon array.
---

Icons renders icon-related `<link>` tags from the build-time configuration resolved by the integration. The component is intentionally thin: file names, MIME types, and inferred `sizes` are computed before the virtual module is serialized, and the component only renders the resolved tags.

When you pass the `icons` prop, it merges over the integration-provided tags for that render. Matching entries are deduplicated by `href`, with later runtime entries overriding earlier build-time entries.

## Props

| prop    | type        | default                                | required | description                                                                                                                      |
| ------- | ----------- | -------------------------------------- | -------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `icons` | `IconTag[]` | integration-resolved icon tags or `[]` | No       | Late-bound icon overrides. Entries are keyed by `href`; runtime entries replace matching build-time entries and append new ones. |

## Usage & Examples

### Basic usage

Input (`.astro`):

```astro
<Icons />
```

Output (`.html`):

```html
<link rel="icon" href="/favicon.ico" type="image/x-icon" />
<link
  rel="apple-touch-icon"
  href="/apple-touch-icon.png"
  sizes="180x180"
  type="image/png"
/>
```

### Auto usage

With an SVG integration source and no explicit `favicon.svg` entry, the integration auto-adds a copied SVG favicon tag to the resolved output.

Input (`.astro`):

```astro
<Icons />
```

Output (`.html`):

```html
<link rel="icon" href="/favicon.svg" sizes="any" type="image/svg+xml" />
<link rel="icon" href="/favicon.ico" type="image/x-icon" />
<link rel="icon" href="/icon-192x192.png" sizes="192x192" type="image/png" />
```

### Complete usage

Input (`.astro`):

```astro
<Icons
  icons={[
    {
      rel: "icon",
      href: "/campaign-dark.png",
      sizes: "32x32",
      type: "image/png",
      media: "dark",
    },
    {
      rel: "preload",
      href: "/campaign-mask.svg",
      as: "image",
      type: "image/svg+xml",
    },
  ]}
/>
```

Output (`.html`):

```html
<link
  rel="icon"
  href="/campaign-dark.png"
  sizes="32x32"
  type="image/png"
  media="(prefers-color-scheme: dark)"
/>
<link rel="preload" href="/campaign-mask.svg" as="image" type="image/svg+xml" />
```

## Decisions Made

### Build-time resolution owns inference

The integration resolves `href`, `type`, and inferred `sizes` before `Icons` renders. This avoids repeating file-extension logic in every page render and keeps the component surface small.

### The runtime prop merges by `href`

Passing `icons` merges over the build-time defaults for that render. Because the build config is keyed by filename, the runtime component uses final `href` values as the override boundary and lets later entries win.

### Light and dark media values expand automatically

When a tag uses `media: "light"` or `media: "dark"`, the component expands that shorthand into the matching `prefers-color-scheme` query. Other custom media strings are passed through unchanged.

### Deduplication is runtime-only and `href`-based

Build-time resolution preserves the raw tag array. `Icons` performs the final dedupe step by `href`, first across the config tags and then across runtime overrides, so later entries replace earlier ones predictably.

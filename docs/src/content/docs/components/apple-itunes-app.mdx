AppleItunesApp renders the `<meta name="apple-itunes-app">` tag that iOS Safari uses to display a Smart App Banner at the top of the browser, linking visitors to an App Store application.

## Props

**Use either `content` OR `id`/`argument`, not both.**

### Option 1: Direct content

| prop      | type     | default | required | description                                                                             |
| --------- | -------- | ------- | -------- | --------------------------------------------------------------------------------------- |
| `content` | `string` | -       | No       | Direct content string for the meta tag. When provided, `id` and `argument` are ignored. |

### Option 2: Build from ID and argument

| prop       | type     | default | required | description                                                              |
| ---------- | -------- | ------- | -------- | ------------------------------------------------------------------------ |
| `id`       | `string` | -       | No       | The App Store application ID. No tag is rendered when omitted.           |
| `argument` | `string` | -       | No       | A URL or deep-link argument passed to the app when the banner is tapped. |

## Examples

### Basic

Input:

```astro
<AppleItunesApp id="123456789" />
```

Output:

```html
<meta name="apple-itunes-app" content="app-id=123456789" />
```

### Automatic (using integration settings)

Uses the info in the integration configuration to render the tag without props.

Input:

```js
// astro.config.mjs
eminence({
  headTags: {
    appleItunesApp: {
      id: "123456789",
      argument: "myapp://open",
    },
  },
});
```

```astro
<AppleItunesApp />
```

Output:

```html
<meta
  name="apple-itunes-app"
  content="app-id=123456789, app-argument=myapp://open"
/>
```

### Complete

Input:

```astro
<AppleItunesApp id="123456789" argument="myapp://open" />
```

Output:

```html
<meta
  name="apple-itunes-app"
  content="app-id=123456789, app-argument=myapp://open"
/>
```

### Direct content

Input:

```astro
<AppleItunesApp content="app-id=123456789, app-argument=myapp://open" />
```

Output:

```html
<meta
  name="apple-itunes-app"
  content="app-id=123456789, app-argument=myapp://open"
/>
```

## Decisions Made

### Two mutually exclusive prop patterns

The component accepts either a `content` prop (for direct control) or `id` and `argument` props (for declarative building). These are mutually exclusive—you cannot use both patterns simultaneously. This enforces a clear contract: either you manage the content string yourself, or you let the component build it for you.

### Only the itunes tag is rendered

This component is scoped exclusively to the `apple-itunes-app` Smart App Banner. Other Apple-specific tags (standalone mode, status bar style, startup images) are not implemented at all. Instead of using: apple-mobile-web-app-title, apple-touch-startup-image, apple-mobile-web-app-capable, apple-mobile-web-app-status-bar-style, etc., you should use the [manifest](/integration/manifest) integration configuration.

### The argument is appended only when provided

The `app-argument` field is omitted from the `content` attribute when no `argument` prop is given. This avoids outputting a malformed or vacuous `app-argument=` entry that could confuse parsers.

### No tag rendered without an id or content

If `id` is not provided and no `content` is given, the component outputs nothing. There is no useful `apple-itunes-app` tag without an App Store application ID or direct content.

Layout renders a full HTML document — `<!doctype html>`, `<html>`, `<head>`, and `<body>` — with `Head` wired in automatically. It is scoped to document shell structure only and does not impose any CSS, styling, or content opinions.

## Props

| prop   | type                          | default | required | description                                                                                                                                        |
| ------ | ----------------------------- | ------- | -------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| `head` | `ComponentProps<typeof Head>` | -       | Yes      | Props forwarded directly to the `Head` component. Accepts all props that `Head` accepts, including `title`, `description`, `canonical`, and so on. |

## Usage & Examples

### Basic usage

Input:

```astro
<Layout head={{ title: "Home" }}>
  <main>Page content</main>
</Layout>
```

Output:

```html
<!doctype html>
<html>
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>Home</title>
  </head>
  <body>
    <main>Page content</main>
  </body>
</html>
```

### Auto usage

Not supported. The `head` prop is required because a document title is always necessary.

### Complete usage

All four slots populated.

Input:

```astro
<Layout head={{ title: "Home", description: "Welcome to my site" }}>
  <Fragment slot="head">
    <link
      rel="preload"
      href="/fonts/body.woff2"
      as="font"
      type="font/woff2"
      crossorigin
    />
  </Fragment>
  <Fragment slot="body-start">
    <div id="skip-nav"><a href="#main">Skip to content</a></div>
  </Fragment>
  <main id="main">Page content</main>
  <Fragment slot="body-end">
    <script src="/app.js"></script>
  </Fragment>
</Layout>
```

Output:

```html
<!doctype html>
<html>
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>Home</title>
    <link
      rel="preload"
      href="/fonts/body.woff2"
      as="font"
      type="font/woff2"
      crossorigin
    />
    <meta name="description" content="Welcome to my site" />
  </head>
  <body>
    <div id="skip-nav"><a href="#main">Skip to content</a></div>
    <main id="main">Page content</main>
    <script src="/app.js"></script>
  </body>
</html>
```

## Decisions Made

### head prop is required

Layout always renders a `Head` component, which itself requires a `title`. Accepting `head` as a required prop makes this dependency explicit and prevents accidental output of a title-less document.

### head slot passes through to Head's own slot

The `Head` component exposes a default slot for extra link and meta tags placed after the title. Layout forwards its `head` slot into that slot, so custom head content remains compatible with the Capo.js ordering `Head` applies.

### Four named regions mirror conventional document structure

The four slots — `head`, `body-start`, default, and `body-end` — match the three conventional body injection points used by analytics scripts, skip-navigation links, deferred scripts, and page content. Using named slots keeps each concern at the correct position without requiring consumers to reproduce the shell markup.

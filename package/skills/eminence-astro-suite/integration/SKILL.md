# Integration features overview

The `eminence()` integration runs in two Astro hooks:

- `astro:config:setup` — registers the Vite virtual module (`virtual:eminence-astro-suite/head-tags`) and the `@astrojs/sitemap` integration.
- `astro:build:done` — writes generated output files (icons, manifest, robots.txt, security.txt) and validates `humans.txt`.

The seven configurable features are summarized below. Read the linked page before configuring each one.

## Features

| Feature               | Option key           | What it does                                                                                                | Read                                   |
| --------------------- | -------------------- | ----------------------------------------------------------------------------------------------------------- | -------------------------------------- |
| Global head defaults  | `headTags`           | Serializes a config object into the Vite virtual module that every tag component imports.                   | [./head-tags.md](./head-tags.md)       |
| Icon generation       | `icons`              | Generates a favicon set + manifest icons from a source image using `sharp` / `sharp-ico`.                   | [./icons.md](./icons.md)               |
| Manifest              | `manifest`           | Writes `dist/manifest.webmanifest` from a typed PWA config. Auto-merges icons from the `icons` feature.     | [./manifest.md](./manifest.md)         |
| robots.txt            | `robotsTxt`          | Writes `dist/robots.txt` with crawler rules and sitemap pointers.                                           | [./robots-txt.md](./robots-txt.md)     |
| security.txt          | `securityTxt`        | Writes `dist/.well-known/security.txt` per RFC 9116.                                                        | [./security-txt.md](./security-txt.md) |
| Sitemap               | `sitemap`            | Registers `@astrojs/sitemap` automatically; options pass through.                                           | [./sitemap.md](./sitemap.md)           |
| humans.txt validation | `headTags.humansTxt` | Validates a user-authored `public/humans.txt` and emits the discovery link. Does **not** generate the file. | [./humans-txt.md](./humans-txt.md)     |

## File outputs at a glance

| File                                                                                                                                               | Generated by                       | Skipped if file already exists  | Disable with                     |
| -------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------- | ------------------------------- | -------------------------------- |
| `dist/favicon.ico`, `dist/favicon.png`, `dist/apple-touch-icon.png`, `dist/icon-192.png`, `dist/icon.png`, `dist/favicon.svg` (when source is SVG) | `icons`                            | partial — per-file logic        | `icons: false`                   |
| `dist/manifest.webmanifest`                                                                                                                        | `manifest`                         | yes (warns)                     | `manifest: false`                |
| `dist/robots.txt`                                                                                                                                  | `robotsTxt`                        | yes (warns)                     | `robotsTxt: false`               |
| `dist/.well-known/security.txt`                                                                                                                    | `securityTxt`                      | yes (warns)                     | `securityTxt: false`             |
| `dist/sitemap-*.xml`                                                                                                                               | `sitemap` (via `@astrojs/sitemap`) | per `@astrojs/sitemap` behavior | `sitemap: false`                 |
| `dist/humans.txt`                                                                                                                                  | **you** (copy from `public/`)      | —                               | (no warning when set to `false`) |

## Resolution and merge order

Several features interact:

1. **`icons` → `headTags.icons` → `<Icons />` overrides**
   - The `icons` integration option generates an `IconTag[]` from the source image.
   - `headTags.icons` is merged over those tags by `href` (replace by key).
   - `<Icons icons={...} />` runtime overrides are merged over the result; `false` removes an entry by `href`.

2. **`icons` → `manifest.icons`**
   - Icons generated with `manifest: true` (or `manifest: ManifestIconOptions`) are added to the manifest `icons` array.
   - `manifest.icons` is merged over the auto-generated set by `src` (replace by key).

3. **`Astro.site` is the root for derived URLs**
   - When `headTags.humansTxt` and `headTags.manifest` are resolved into absolute URLs, `Astro.site` is used as the base.
   - `Canonical` and `OpenGraph` also derive URLs from `Astro.site` at render time.

## Recommendation warnings

The integration emits build-time recommendations when an optional output is **omitted** (not set to `false`):

- Missing `humans.txt` → recommend reading [../recommendations/humans-txt.md](../recommendations/humans-txt.md).
- Missing `robotsTxt` → recommend reading [../recommendations/robots-txt.md](../recommendations/robots-txt.md).
- Missing `securityTxt` → recommend reading [../recommendations/security-txt.md](../recommendations/security-txt.md).
- Missing `manifest` → recommend reading [../recommendations/manifest.md](../recommendations/manifest.md).

Pass `false` (instead of omitting) to silence the warning when you have intentionally decided not to add the output.

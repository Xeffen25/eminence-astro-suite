---
title: Icons - Component
description: Documentation for Icons, focused on rendering resolved icon link tags. It accepts a late-bound replacement array when a page needs custom icon output.
---

Icons renders icon-related `<link>` tags from the build-time configuration resolved by the integration. The component is intentionally thin: file names, MIME types, and inferred `sizes` are computed before the virtual module is serialized, and the component only renders the resolved tags.

When you pass the `icons` prop, it replaces the integration-provided tags for that render. This is useful for page-specific campaigns, experiments, or other late-bound overrides.

## Props

| prop    | type        | default                                | required | description                                                                                                                                  |
| ------- | ----------- | -------------------------------------- | -------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| `icons` | `IconTag[]` | integration-resolved icon tags or `[]` | No       | Complete late-bound replacement list for icon `<link>` tags. Duplicate tags are deduplicated by `rel`, `href`, `sizes`, `type`, and `media`. |

## Usage & Examples

### Basic usage

Input (`.astro`):

```astro
<Icons />
```

Output (`.html`):

```html
<link rel="icon" href="/favicon.ico" type="image/x-icon" />
<link rel="apple-touch-icon" href="/apple-touch-icon.png" sizes="180x180" type="image/png" />
```

### Auto usage

With an SVG integration source and no explicit `favicon.svg` entry, the integration auto-adds a copied SVG favicon tag to the resolved output.

Input (`.astro`):

```astro
<Icons />
```

Output (`.html`):

```html
<link rel="icon" href="/favicon.svg" sizes="any" type="image/svg+xml" />
<link rel="icon" href="/favicon.ico" type="image/x-icon" />
<link rel="icon" href="/icon-192x192.png" sizes="192x192" type="image/png" />
```

### Complete usage

Input (`.astro`):

```astro
<Icons
	icons={[
		{ rel: "icon", href: "/campaign-dark.png", sizes: "32x32", type: "image/png", media: "dark" },
		{ rel: "preload", href: "/campaign-mask.svg", as: "image", type: "image/svg+xml" },
	]}
/>
```

Output (`.html`):

```html
<link rel="icon" href="/campaign-dark.png" sizes="32x32" type="image/png" media="(prefers-color-scheme: dark)" />
<link rel="preload" href="/campaign-mask.svg" as="image" type="image/svg+xml" />
```

## Decisions Made

### Build-time resolution owns inference

The integration resolves `href`, `type`, and inferred `sizes` before `Icons` renders. This avoids repeating file-extension logic in every page render and keeps the component surface small.

### The runtime prop is a full replacement

Passing `icons` replaces the build-time defaults for that render instead of merging by filename. At render time only the final `IconTag[]` remains, so full replacement is the clearest and least ambiguous override model.

### Light and dark media values expand automatically

When a tag uses `media: "light"` or `media: "dark"`, the component expands that shorthand into the matching `prefers-color-scheme` query. Other custom media strings are passed through unchanged.

### Deduplication remains content-based

Tags are deduplicated by `rel`, `href`, `sizes`, `type`, and `media`. This keeps repeated runtime arrays or duplicated build output from rendering duplicate `<link>` elements.

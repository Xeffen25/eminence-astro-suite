ColorScheme renders a `<meta name="color-scheme">` tag that tells the user agent which page color schemes are supported. It is limited to serializing the configured `content` value and does not infer schemes from CSS or page styling.

## Props

| prop      | type                                                                            | default                        | required | description                                                |
| --------- | ------------------------------------------------------------------------------- | ------------------------------ | -------- | ---------------------------------------------------------- |
| `content` | `"normal" \| "light" \| "dark" \| "light dark" \| "dark light" \| "only light"` | `clientHeadConfig.colorScheme` | No       | Supported page color schemes advertised to the user agent. |

## Usage & Examples

### Basic usage

Input:

```astro
<ColorScheme content="light dark" />
```

Output:

```html
<meta name="color-scheme" content="light dark" />
```

### Auto usage

Uses the integration `head.colorScheme` value when no prop is provided.

Input:

```astro
<ColorScheme />
```

Output:

```html
<meta name="color-scheme" content="light dark" />
```

### Complete usage

Input:

```astro
<ColorScheme content="only light" />
```

Output:

```html
<meta name="color-scheme" content="only light" />
```

## Decisions Made

### The component renders only when content is available

ColorScheme emits no tag when neither the `content` prop nor the integration configuration provides a value. This avoids outputting an incomplete meta tag with no declared scheme.

### Supported values are constrained to known color-scheme tokens

The `content` prop accepts a fixed union of supported color-scheme values. This keeps usage aligned with the values understood by the meta tag and reduces invalid input at the type level.

### Integration config can provide the default

When `content` is omitted, ColorScheme falls back to `clientHeadConfig.colorScheme`. This supports defining a site-wide default once while still allowing page-level overrides.

The `eminence` integration is configured through a single options object passed to `createIntegration`. Each key maps to a distinct feature. All features are optional and can be disabled by passing `false`. Omitting a feature key logs a recommendation warning where applicable.

```ts
// astro.config.mjs
import { defineConfig } from "astro/config";
import eminence from "eminence-astro-suite";

export default defineConfig({
	site: "https://example.com",
	integrations: [
		eminence({
			head: { ... },
			icons: { ... },
			manifest: { ... },
			robotsTxt: { ... },
			securityTxt: { ... },
			sitemap: { ... },
		}),
	],
});
```

## `head`

Controls default values available to `<head>` components via the virtual config module. All fields are optional.

| option              | type                     | description                                                     |
| ------------------- | ------------------------ | --------------------------------------------------------------- |
| `charset`           | `string`                 | Default charset for the `Charset` component.                    |
| `viewport`          | `string`                 | Default viewport content string for the `Viewport` component.   |
| `base`              | `object`                 | Props forwarded to the `Base` component.                        |
| `colorScheme`       | `string`                 | Default color scheme for the `ColorScheme` component.           |
| `titleTemplate`     | `string`                 | Title template string for the `Title` component.                |
| `appleItunesApp`    | `object`                 | Props forwarded to the `AppleItunesApp` component.              |
| `appLinks`          | `object`                 | Props forwarded to the `AppLinks` component.                    |
| `generator`         | `boolean`                | Whether to render the `Generator` tag.                          |
| `openGraphSiteName` | `string`                 | Site name for Open Graph tags.                                  |
| `robots`            | `object`                 | Props forwarded to the `Robots` component.                      |
| `themeColor`        | `object`                 | Props forwarded to the `ThemeColor` component.                  |
| `humansTxt`         | `string \| URL \| false` | URL for the `<link rel="author">` tag. Pass `false` to opt out. |
| `verification`      | `object`                 | Props forwarded to the `Verification` component.                |

## `icons`

Generates favicon and app icon files at build time from a single source image. See [Icons](/integration/icons) for full option details.

| option             | type                   | description                                                                |
| ------------------ | ---------------------- | -------------------------------------------------------------------------- |
| `source`           | `string \| false`      | Path to the source image. Pass `false` to skip generation without warning. |
| `overrides`        | `IconsOverrides`       | Replace or disable individual default icons.                               |
| `customGeneration` | `GeneratedIconTag[]`   | Additional icons to generate at custom sizes and formats.                  |
| `customTags`       | `IconTag[]`            | Extra link tags to inject without generating files.                        |
| `manifest`         | `IconsManifestOptions` | Controls which generated icons are contributed to the web manifest.        |

Pass `icons: false` to disable generation entirely with no warning.

## `manifest`

Generates a `/manifest.webmanifest` file at build time. Chromium PWA requirements (`name`/`short_name`, `display`/`display_override`, `start_url`) are enforced as TypeScript types. See [manifest.webmanifest](/integration/manifest) for full option details.

| option             | type                    | required | description                                                              |
| ------------------ | ----------------------- | -------- | ------------------------------------------------------------------------ |
| `name`             | `string`                | Yes\*    | Full app name. Required unless `short_name` is provided.                 |
| `short_name`       | `string`                | Yes\*    | Short app name. Required unless `name` is provided.                      |
| `start_url`        | `string`                | Yes      | URL loaded when the app is launched.                                     |
| `display`          | `string`                | Yes\*\*  | Display mode. Required unless `display_override` is provided.            |
| `display_override` | `string[]`              | Yes\*\*  | Ordered display mode candidates. Required unless `display` is provided.  |
| `icons`            | `WebManifestIconItem[]` | No       | Icons for the app. Auto-populated from the icons integration if omitted. |
| _(more)_           |                         |          | Additional optional fields documented on the manifest page.              |

Pass `manifest: false` to disable generation with no warning.

## `robotsTxt`

Generates a `/robots.txt` file at build time. If a `robots.txt` already exists in the build output, generation is skipped. See [robots.txt](/integration/robots-txt) for full option details.

| option    | type                               | required | description                                                      |
| --------- | ---------------------------------- | -------- | ---------------------------------------------------------------- |
| `rules`   | `RobotsTxtRule \| RobotsTxtRule[]` | Yes      | One or more crawl rules defining agent access.                   |
| `sitemap` | `string \| string[]`               | No       | Absolute URL(s) or site-relative path(s) for `Sitemap:` entries. |

Pass `robotsTxt: false` to disable generation with no warning.

## `securityTxt`

Generates a `/.well-known/security.txt` file at build time following [RFC 9116](https://www.rfc-editor.org/rfc/rfc9116). See [security.txt](/integration/security-txt) for full option details.

| option               | type                                           | required | description                                                                    |
| -------------------- | ---------------------------------------------- | -------- | ------------------------------------------------------------------------------ |
| `contact`            | `string \| string[]`                           | Yes      | `mailto:` address(es) or `https://` URL(s).                                    |
| `expires`            | `Date \| string \| SecurityTxtExpiresDuration` | Yes      | Policy expiry. Accepts a `Date`, ISO 8601 string, or duration like `"1 year"`. |
| `encryption`         | `string \| string[]`                           | No       | HTTPS URL(s) to a PGP key.                                                     |
| `policy`             | `string \| string[]`                           | No       | HTTPS URL(s) to the security disclosure policy.                                |
| `preferredLanguages` | `string \| string[]`                           | No       | BCP 47 language tag(s) for preferred report language(s).                       |
| _(more)_             |                                                |          | Additional optional fields documented on the security.txt page.                |

Pass `securityTxt: false` to disable generation with no warning.

## `sitemap`

Delegates sitemap generation to [`@astrojs/sitemap`](https://docs.astro.build/en/guides/integrations-guide/sitemap/), which must be installed as a peer dependency. The integration is registered automatically — no need to add it separately. See [Sitemap](/integration/sitemap) for full option details.

The `sitemap` option accepts a `SitemapOptions` object (forwarded directly to `@astrojs/sitemap`) or `false`.

Omitting `sitemap` is equivalent to `sitemap: {}` — enabled with default options.

Pass `sitemap: false` to disable the feature and suppress the peer dependency requirement.

## Full example

```ts
// astro.config.mjs
import { defineConfig } from "astro/config";
import eminence from "eminence-astro-suite";

export default defineConfig({
	site: "https://example.com",
	integrations: [
		eminence({
			head: {
				charset: "UTF-8",
				viewport: "width=device-width, initial-scale=1",
				titleTemplate: "%s | My Site",
				colorScheme: "light dark",
				humansTxt: "https://example.com/humans.txt",
			},
			icons: {
				source: "./src/assets/icon.svg",
			},
			manifest: {
				name: "My Site",
				short_name: "MySite",
				start_url: "/",
				display: "standalone",
				background_color: "#ffffff",
				theme_color: "#0070f3",
			},
			robotsTxt: {
				rules: { agent: "*", disallow: "/private/" },
				sitemap: "/sitemap-index.xml",
			},
			securityTxt: {
				contact: "mailto:security@example.com",
				expires: "1 year",
				policy: "https://example.com/security-policy",
			},
			sitemap: {},
		}),
	],
});
```

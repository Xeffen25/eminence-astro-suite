Eminence Astro Suite is both:

- a component library for metadata generation, and
- an Astro integration that powers those components through a Vite virtual module and automates important output files.

For most components, the integration must be configured in `astro.config.mjs` so runtime defaults and derived metadata are available, and project outputs like `sitemap.xml`, `security.txt`, and `robots.txt` can be generated consistently.

## How to use it

<Steps>

1. Add the integration in your Astro config.

    ```ts title="astro.config.mjs"
    import { defineConfig } from "astro/config";
    import eminence from "eminence-astro-suite";

    export default defineConfig({
    	integrations: [
    		eminence({
    			// Your project defaults for metadata and generated files.
    		}),
    	],
    });
    ```

2. Use only the `Head` component in pages/layouts.

3. Start with minimal props (`title`, `description`) and scale up with advanced props like JSON-LD, Open Graph, and other metadata surfaces as needed.

</Steps>

## Why this design

The intended workflow is simple and opinionated:

- Configure integration once.
- Use one entry point (`Head`) everywhere.
- Move from basic SEO to advanced metadata without changing architecture.

This keeps metadata consistent across pages, improves DX, and helps prevent mistakes caused by manual and fragmented tag composition.

## Integration-powered outputs

Beyond metadata tags, the integration can automate related outputs:

- icon generation (favicons and manifest-oriented icon sets)
- `security.txt`
- `robots.txt`
- `sitemap.xml` (using `@astrojs/sitemap` under the hood)
- runtime `<head>` reordering with [Capojs integration](/integration/capojs)

## Scope and relevance

Eminence targets relevant, modern SEO/metadata concerns and intentionally avoids obsolete surfaces.

- Unsupported file outputs are documented in [Unsupported Files](/policies/unsupported-files).
- Unsupported tags are documented in [Unsupported Tags](/policies/unsupported-tags).
